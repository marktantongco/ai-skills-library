Operating Principle

How this library works — build systems, not one-off tasks.

⚡

No One-Off Work

Every repeatable request becomes a skill. If you ask for the same thing twice, the system failed the first time.

🔍

Check First

Search existing skills before creating new ones. Extend, don't duplicate. MECE discipline: one owner skill per type of work.

🚀

Build It Once

Prototype, get approval, codify into SKILL.md, schedule on cron. Every skill added makes the system smarter.

🛡

Security First

Every skill is vetted before installation. No arbitrary code execution, no unauthorized access, no suspicious patterns.

Getting Started

How to use and extend this skills library.

📁

Install Skills

Copy any skill's SKILL.md into your agent's skills directory. Each skill has 4 sections: context, instructions, constraints, and examples.

🔎

Discover More

Use the Skill Finder meta-skill to search skills.sh, GitHub, and Anthropic Academy for new capabilities.

🔧

Create Your Own

Follow the SKILL.md format: context (when to use), instructions (step-by-step), constraints (hard rules), examples (ideal output).

📈

Compounding System

Every skill added makes the system smarter. Schedule audits, extend existing skills, and never duplicate coverage.
